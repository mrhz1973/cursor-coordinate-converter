1. **Scope bounded** — CoT XML ha schema pubblico e stabile (MITRE). Import ed export sono separabili; v1 può essere unidirezionale (export prima, import dopo) come esplicitamente consentito dal roadmap.

4. **Validazione metodo** — valida il workflow dev-method v0.1.1 su una feature non banale ma controllata, prima di impegnarsi su lavoro più ampio (compound polygon).
5. **Allineamento roadmap** — chiude T1.2 esplicitamente listato come gap tattico core Tier 1.

**Sequenza consigliata**:
- **Blocco 1 (ora)**: CoT XML import/export (T1.2)
- **Blocco 2 (dopo CoT PASS)**: Compound polygon — pianificare come multi-step pass dedicato
- **Separato**: PCN/Geoportale — sessione research gate-clearance prima di qualsiasi implementazione
