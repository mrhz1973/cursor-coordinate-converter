**Data:** 2026-05-01 00:05  
**Progetto:** APP GIS / GOI GIS Tool

---

## Identità del progetto

Il progetto è una app GIS tattica leggera, offline-first e OPSEC-aware, evoluta da convertitore coordinate verso uno strumento operativo GIS compatto.

File operativo principale:

```text
coordinate_converter Claude.html
```

Deliverable vincolante:

```text
un singolo file HTML standalone
```

L’app deve poter funzionare come file unico, senza build step e senza struttura multi-file runtime.

---

## Vincoli architetturali permanenti

Non proporre e non far implementare:

- React, Vue, Svelte, Lit o altri framework;
- TypeScript;
- npm;
- bundler;
- transpiler;
- ES modules;
- split operativo del deliverable;
- package.json per l’app;
- dipendenze runtime esterne non necessarie;
- riscritture architetturali ampie senza decisione esplicita.

Stack previsto:

```text
HTML + CSS + JavaScript vanilla nello stesso file
```

---

## Principi funzionali importanti

L’app è:

- GIS-first;
- offline-first;
- OPSEC-aware;
- pensata per uso anche in contesti senza rete;
- orientata a mappa centrale e pannelli floating ordinati.

Regole importanti:

- nessun GPS silenzioso all’avvio;
- geolocation solo su azione esplicita;
- niente live tracking GPS reintrodotto senza decisione esplicita;
- OPSEC strict deve bloccare chiamate di rete sensibili;
- geocoding online solo con opt-in e nel rispetto OPSEC/offline;
- tile online e geocoding non devono diventare dipendenze obbligatorie;
- cache offline e tile scaricate non vanno cancellate da azioni metadata-only.

## Moduli e aree funzionali principali

L’app include o sta includendo:

- conversione coordinate DD, DDM, DMS, UTM, MGRS, Plus Code;
- datum italiani ed extra;
- mappa centrale GIS;
- tile online/offline;
- Mappe Offline e aree salvate;
- waypoint manager;
- track builder;
- misure su mappa;
- favoriti;
- cronologia;
- geocoding;
- export/import formati GIS;
- Range Rings / centri concentrici;
- pannelli floating in GIS mode.

## Regole UI generali consolidate

L’utente ha chiesto coerenza UI tra modal/pannelli:

- pannelli floating ordinati e non invasivi;
- X/close uniforme;
- pulsanti coerenti tra modal;
- primary solo per azione primaria reale;
- azioni secondarie come secondary/ghost/subtle;
- danger per eliminazioni;
- liste uniformi con visibilità, edit, delete e data;
- conferme interne, niente `confirm` nativo;
- contenuto e azioni principali in alto;
- opzioni/toggle secondari in basso;
- pannelli trascinabili, possibilmente anche parzialmente fuori mappa, ma sempre recuperabili.

## Nota su Range Rings

Range Rings è lo strumento corrente in lavorazione.

Obiettivo utente più ampio:

- pulsante Rings mappa più leggibile;
- toggle apri/chiudi;
- UI coerente con gli altri pannelli;
- lista uniforme;
- import/export non primary se non serve;
- default distanze modificabile;
- in futuro spostamento del gruppo anelli sulla mappa e drag avanzato.

Il drag avanzato del ring sulla mappa è da trattare in un blocco successivo, non nel Blocco 1 UI.
