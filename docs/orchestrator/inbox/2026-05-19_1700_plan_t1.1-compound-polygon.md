### Geometry sanitizer: comportamento chiave

`gisSanitizeGeometry(geometry, "polygon")`:
- Accetta `type: "Polygon"` con outer ring in `coordinates[0]`.
- Deduplicazione coordinate adiacenti.
- Cap 500 punti per ring.
- Richiede minimo 3 vertici distinti.
- Auto-chiude il ring (aggiunge `ring[0]` alla fine se non già chiuso).
- Ignora holes (solo `coordinates[0]`).

### Properties sanitizer: campi disponibili

```js
{
  name:        string  max 80
  description: string  max 500
  category:    string  max 40
  symbol:      string  max 40
  source:      string  max 80
  status:      string  max 40
  priority:    string  max 20
  notes:       string  max 500
  color:       string  sanitizeTrackColor()
  size:        number  [0.25, 6]
  visible:     bool    default true
  locked:      bool    default false
  kind:        string  "polygon"
  created_at:  string  ISO8601
  updated_at:  string  ISO8601
}
```

## 3. Modello entità definitivo

Il modello è già definito dalla struttura GeoJSON Feature interna:

```js
{
  id:         string   // uid generato
  type:       "Feature"
  geometry: {
    type:         "Polygon"
    coordinates:  [[[lon,lat], ...]]  // ring chiuso, max 500 vertici
  }
  properties: {
    name:        string  // nome operativo (es. "Area Alfa")
    description: string  // descrizione estesa
    category:    string  // "operational-area" | "exclusion-zone" | "grid" | custom
    color:       string  // colore fill/stroke
    notes:       string  // note operative
    visible:     bool
    locked:      bool
    kind:        "polygon"
    created_at:  ISO8601
    updated_at:  ISO8601
  }
  layerId: string  // default GIS_DEFAULT_LAYER_IDS.polygons
}
```

**Cap:** 50 poligoni, 500 vertici per poligono (già hardcoded nei sanitizer).

**Raccomandazione: pannello floating dedicato `#polygonPanel`**, seguendo il pattern consolidato di RR/Astro/Misura.

| Opzione | Pro | Contro | Decisione |
|---------|-----|--------|-----------|

## 5. Interazione mappa

### Fasi in scope per v1

1. **Crea modalità** — utente clicca "Nuovo poligono" → app entra in polygon-draw mode (cursore crosshair, hint visivo, simile a `waypointPickMode`).
2. **Click-to-add vertex** — ogni click sulla mappa aggiunge un vertice. Pattern: `onUp` handler, simile a track pick / waypoint pick. Draft polygon visualizzato in tempo reale.
3. **Chiudi poligono** — pulsante "Chiudi poligono" nel pannello (o doppio-click sull'ultimo vertice) → `gisSanitizeGeometry` valida, `gisFeatureAdd("polygon", ...)`, re-render.
4. **Annulla** — Esc o pulsante "Annulla" → scarta draft vertices, esce da polygon-draw mode.
5. **Edit nome/note** — editor inline nel pannello dopo chiusura (simile a waypoint editor).
6. **Seleziona esistente** — click sulla lista nel pannello → highlight sulla mappa.
7. **Elimina** — pulsante 🗑 con conferma interna (pattern `#wpDeleteOneBar` / `#trackSavedDeleteBar`).

### Fuori scope v1 (deferred)

- Drag vertex per spostamento.
- Edit geometria post-chiusura.
- Import polygon da file.
- Hole / MultiPolygon.
- CoT polygon interop.

state.polygonDrawMode = false    // in draw mode
