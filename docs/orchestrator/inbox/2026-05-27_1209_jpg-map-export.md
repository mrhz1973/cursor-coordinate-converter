### Logica JS

Nuove funzioni nel monolite (vicino a `downloadBlob`):

- `formatJpgExportTimestamp(d)` — produce `YYYYMMDD-HHMMSS` locale per il filename.
- `rasterizeSvgOntoCanvas(svgEl, ctx, dx, dy, dw, dh)` — serializza un nodo SVG con `XMLSerializer`, lo carica via `Blob` + `URL.createObjectURL` in un `<img>` e lo disegna su un `CanvasRenderingContext2D`.
- `exportMapAsJpg()` — pipeline principale:
  1. trova `#miniMap > .tile-map` (se assente → `showError(t("export.jpg.noMap"))` e abort);
  2. crea canvas delle dimensioni del viewport mappa, riempie con bianco;
  3. itera su `img.tile` completi e con `naturalWidth > 0`, calcola posizioni con `getBoundingClientRect` rispetto al `.tile-map` e fa `drawImage` — i tile sono `blob:` URL (impostati da `hydrateMapTiles` da Blob same-origin via `fetch` CORS o IndexedDB) → canvas **non tainted**;
  4. raster di `.tile-grid` SVG (overlay MGRS/UTM) se presente;
  5. raster di tutti gli `.tile-marker svg` (pin del punto convertito + eventuali draft);
  6. `canvas.toBlob(cb, "image/jpeg", 0.92)` → download `gis-map-export-YYYYMMDD-HHMMSS.jpg` tramite `<a download>` e `exportPrefixedDownloadFilename`.
- Catch su `SecurityError` / `tainted`: mostra `t("export.jpg.tainted")`; altri errori → `t("export.jpg.fail")`. Mai `alert()` / `window.confirm()`.

Click handler legato in init UI subito dopo quello di `btnPrint` (sezione standard listener).

## Limiti / rischi residui

- **Canvas tainted (CORS):** mitigato — i tile usano `blob:` URL same-origin (sempre via `hydrateMapTiles`), quindi `toBlob()` dovrebbe sempre riuscire. Se un tile fosse caricato direttamente con `<img src="https://...">` senza `crossorigin`, il canvas verrebbe tainted; in quel caso l'errore viene catturato e mostrato con `t("export.jpg.tainted")`.
- **Overlay non rasterizzati:** controlli flottanti (zoom, layers menu, measure box, track float, waypoint editor, attribution overlay) **non** vengono inclusi nell'export — solo tile + griglia + marker pin. Coerente con "export semplice della vista mappa".
- **Risoluzione:** è quella del viewport DOM, non DPI alta. Per stampa A4 può apparire morbida — accettabile per "snapshot / print semplice".
- **GeoTIFF / georeferenziazione raster:** non implementati (out of scope per task futuri).
- **VPS staging:** non aggiornato (out of scope esplicito).
