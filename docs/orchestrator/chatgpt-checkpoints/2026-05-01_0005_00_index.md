## Cosa deve capire subito una nuova chat

1. L’app è **GOI GIS Tool / APP GIS**.
2. Il file operativo principale è:

```text
coordinate_converter Claude.html
```

3. L’architettura è vincolante:

```text
single-file HTML standalone
HTML/CSS/JS nello stesso file
vanilla JavaScript
no framework
no TypeScript
no npm
no bundler
no ES modules
no split operativo
```

4. ChatGPT è il coordinatore operativo: ragiona, controlla GitHub con `aggio`, prepara prompt Cursor, valuta risultati.
5. Cursor è l’implementatore: modifica il monolite e aggiorna docs quando richiesto.
6. GitHub è la memoria remota leggibile da ChatGPT.
7. Il comando `finito` è da dare a Cursor, non a ChatGPT.
8. Il comando `checkpoint` crea memoria ChatGPT su GitHub.
9. Il comando `checkpoint esteso` crea questo tipo di bundle dettagliato.
10. Il comando `aggio` o `aggiornati` significa: ChatGPT deve leggere GitHub, in particolare `docs/orchestrator/latest.md`.
